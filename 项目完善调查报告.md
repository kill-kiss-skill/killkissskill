# ITAS 项目完善调查报告

> 审计范围：前端 19 个页面 + 后端 9 个控制器 + 7 个服务 + RAG引擎  
> 审计日期：2026-05-09

---

## 一、严重问题（安全 & 功能缺陷）

### 1.1 前端无路由守卫（安全）

**问题**：`router/index.js` 没有 `beforeEach` 导航守卫。任何已登录用户在地址栏输入 `/admin` 即可访问管理员页面，侧边栏仅做了视觉隐藏，不构成安全防护。

**影响**：学生可直接访问教师/管理员页面，查看其他学生数据。

### 1.2 后端多处缺少 @PreAuthorize（安全）

| 控制器 | 问题 |
|--------|------|
| `KnowledgeController` | 全部方法无角色检查，任何登录用户可上传/删除文档 |
| `ChatController` | 无角色检查 |
| `LearningController` | `saveScore`、`importScores` 无教师/管理员限制 |
| `ScoreController` | `studentScores`、`analyze` 可查看任意学生成绩 |

### 1.3 API 密钥明文存储（安全）

`application-dev.yml` 中 DeepSeek API Key、智谱AI API Key、Milvus Token 全部明文硬编码，提交到 Git 有泄露风险。

### 1.4 ANALYZE Agent 死代码（功能 BUG）

`ChatServiceImpl.java` 第 72-77 行：当 MasterAgent 路由到 `ANALYZE` 时，代码回退到 `chatAgent.answer()`，**从未调用** `LearningAnalyzeAgent`。AI 成绩分析功能实际走的是通用对话路径，分析专用 Agent 是死代码。

### 1.5 教师无法查看学生分析（功能 BUG）

`teacher/students.vue` 点击"查看详情"导航到 `/student/progress?studentId=X`，但 `progress.vue` 使用的是 `store.getters['user/currentUser'].id`（教师自己的ID），而非 URL 参数中的 `studentId`。教师永远看不到学生的成绩分析。

### 1.6 学习计划生成无结构化项目（功能缺陷）

`LearningServiceImpl.generatePlan()` 调 AI 生成文本存入 `content`，但**从未在 `learning_plan_item` 表创建条目**。前端的计划勾选切换功能 (`togglePlanItem`) 因此没有数据可操作。

### 1.7 聊天历史时间戳 BUG

`ChatBox.vue` `loadHistory()` 将所有历史消息的时间设为 `formatDate(new Date())`（当前时间），忽略后端返回的真实时间戳。

## 二、中等优先级问题

### 2.1 Toast 系统完全未使用

`Toast.vue` 已创建、`showToast()` 已导出、`App.vue` 已挂载，但**所有页面均未调用**。错误/成功反馈仍用局部变量或 `console.error`。

### 2.2 DataTable 组件完全未使用

`DataTable.vue` 提供了骨架屏、空状态、分页、列插槽等功能，但**没有任何页面引用它**。每个列表页仍各自内联表格样式。

### 2.3 EmptyState 组件仅 2 处使用

10+ 个页面显示"暂无数据"时用的是纯文字，而非带图标的 `EmptyState` 组件。

### 2.4 12 个列表页中 8 个无分页

| 有分页 | 无分页 |
|--------|--------|
| admin/users、admin/logs | student/knowledge、student/resources、student/learning、teacher/students、teacher/documents、admin/departments、admin/classes、admin/courses |

### 2.5 全局静默错误处理

几乎所有页面的 API 调用使用 `catch (_) {}` 或 `console.error(...)`，用户看不到任何错误提示。共涉及 10+ 个页面。

### 2.6 设计系统混用

| 已迁移到新设计系统 | 仍用旧样式 |
|-------------------|-----------|
| 3 个仪表盘、课程管理、班级管理、ChatBox、Layout/NavBar/SideBar | knowledge、learning、progress、resources、scores、documents、students、users、departments、logs |

旧页面仍硬编码 `#333`、`#999`、`#409EFF`，未用 CSS 变量。

### 2.7 后端分页返回不完整

`AdminController.listUsers` 返回 `List` 而非 `Page`，前端无法获取总条数和总页数。

### 2.8 仪表盘 N+1 查询

`DashboardServiceImpl.getTeacherStudents` 对每个学生循环执行 3 次独立查询（100 学生 = 300 次 DB 查询）。`getStudentDashboard` 在循环内逐条查课程名称。

## 三、缺失的功能

### 3.1 管理员端

| 缺失功能 | 说明 |
|---------|------|
| 编辑用户 | 无法修改用户姓名/邮箱/电话/角色 |
| 重置密码 | 无法帮用户重置密码 |
| 删除用户 | 无法删除用户 |
| 编辑院系 | 院系只能增删，不能改名 |
| 无 404 页面 | 未定义路由显示空白页 |

### 3.2 教师端

| 缺失功能 | 说明 |
|---------|------|
| 编辑/删除成绩 | 录入后无法修改或删除 |
| 按课程查看成绩 | 无法按课程分组查看全班成绩 |
| 查看学生计划 | 无法查看 AI 给学生生成的学习计划 |
| 导入错误详情 | Excel 导入后只显示批次号，不显示哪些行失败 |

### 3.3 学生端

| 缺失功能 | 说明 |
|---------|------|
| 知识库搜索 | 只能按学科筛选，无关键词搜索 |
| 知识库预览 | 无法预览文档内容 |
| 资源搜索/筛选 | 无法搜索或按难度筛选资源 |
| 移动端适配 | 仅仪表盘有响应式，其他页面无 |

### 3.4 通用

| 缺失功能 | 说明 |
|---------|------|
| 聊天 SSE 流式输出 | AI 对话是阻塞式请求-响应 |
| 聊天速率限制 | AI API 无任何调用频率限制 |
| JWT 令牌撤销 | 退出登录后 Token 仍然有效 |
| Redis 缓存 | Redis 已配置但从未使用 |
| 单元测试 | `src/test/` 完全为空 |
| 全局异常处理 | 缺少 8 种常见异常的处理（404/类型转换/外键约束等） |
